### الخطوة 2️⃣: إضافة البيانات الأساسية

```powershell
sqlcmd -S "(localdb)\mssqllocaldb" -d "SmartAccountant_v1005_DB" -f 65001 -i "DATA_COMPLETE.sql"
```

> ملاحظة مهمة: استخدام `-f 65001` يحافظ على النص العربي داخل السكربتات ويمنع ظهور أحرف مشوهة مثل `Ø§Ù...`.

**يتم إضافة:**
- ✅ 5 عملات (SAR, AED, EGP, USD, EUR)
- ✅ 3 خطط اشتراك
- ✅ 4 أدوار نظام
- ✅ 5 صلاحيات
- ✅ 20 دولة عربية
- ✅ حساب إدارة + مستخدم مسؤول
- ✅ 6 إعدادات نظامية

## ملاحظات مهمة ⚠️

### 🌍 دعم اللغة العربية
- جميع الملفات تستخدم حرف `N` قبل النصوص العربية: `N'الكلمة العربية'`
- قاعدة البيانات تستخدم الجودة `Arabic_100_CI_AS` للدعم الكامل
- جميع الحقول النصية هي `NVARCHAR` وليس `VARCHAR`

### 🔐 الأمان
- كلمة المرور الإدارية مشفرة بـ bcrypt
- **غيّر كلمة المرور بعد الإعداد الأول!**

### ♻️ إعادة التشغيل
إذا أردت إعادة تشغيل البيانات من جديد:
```powershell
# 1. نظّف البيانات
sqlcmd -S "(localdb)\mssqllocaldb" -d "SmartAccountant_v1005_DB" -i "00_Cleanup_Corrupted_Data.sql"

# 2. أعد إدراج البيانات
sqlcmd -S "(localdb)\mssqllocaldb" -d "SmartAccountant_v1005_DB" -i "01_Insert_Base_Data.sql"
```

## المشاكل الشائعة

### مشكلة: "Database already exists"
**الحل**: إذا كانت قاعدة البيانات موجودة بالفعل، ستُنشأ الجداول الإضافية فقط.

### مشكلة: "Collation not supported"
**الحل**: تأكد من نسخة SQL Server وأنها تدعم `Arabic_100_CI_AS`

### مشكلة: النصوص العربية مشوهة
**الحل**: تأكد من تشغيل جميع الملفات والالتزام بحرف `N` للنصوص
